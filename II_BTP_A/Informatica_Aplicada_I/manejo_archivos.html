¿Qué es un Sistema de Archivos?

Es la estructura lógica y organizada que utiliza el sistema operativo (como Windows) para almacenar, organizar, nombrar, manipular y proteger los archivos y datos en un dispositivo de almacenamiento.

El Explorador de Archivos de Windows

Es la principal interfaz gráfica para acceder y gestionar el sistema de archivos. Permite navegar entre directorios, crear carpetas, mover, copiar y eliminar archivos de manera intuitiva.

Propiedades y Atributos

Propiedades (Metadatos)

Información detallada sobre el archivo (tamaño, fecha de creación/modificación, autor, tipo, etc.). Accesible al hacer clic derecho y seleccionar "Propiedades".

Atributos (Comportamiento)

Marcadores que definen cómo el sistema operativo debe manejar el archivo.

  • Solo lectura: Impide modificar o eliminar accidentalmente.
  • Oculto: No se muestra en la vista normal del explorador.
  • De Sistema: Archivos esenciales del OS, protegidos.

Software de Compresión: ¿Por qué y Cómo?

La compresión reduce el tamaño de los archivos para ahorrar espacio o facilitar su transferencia (ej. envío por email).

Tipos Comunes

  • .ZIP: Estándar integrado en Windows.
  • .RAR: Formato propietario de WinRAR.
  • .7Z: Formato de alta compresión de 7-Zip.

Compresión sin pérdida (Lossless)

Permite reconstruir el archivo original exactamente igual después de la descompresión. Es ideal para documentos y código.

Montaje de Disco Virtual

Consiste en hacer que un archivo de imagen (como un .ISO o .MDS) se comporte como una unidad de disco física (CD/DVD/Blu-ray) en el sistema.

ESPACIO PARA IMAGEN: Icono o logo de Daemon Tools o un disco virtual.
Software Ejemplo: Daemon Tools, Virtual CloneDrive o la función nativa de Windows 10/11 para ISOs.

Gestión de Respaldo (Backup)

El respaldo es la copia de datos importantes para protegerlos contra pérdidas por fallas de hardware, errores humanos, o ataques cibernéticos (ej. ransomware).

Respaldo Manual

El usuario copia y pega archivos a un destino externo cuando lo recuerda.

Respaldo Automático

El sistema o software programado realiza la copia periódicamente sin intervención del usuario (ej. Historial de Archivos de Windows).

Estrategia 3-2-1 para Backups

Una regla de oro en la gestión de datos para maximizar la seguridad.

Regla 3-2-1

  • 3 Copias: Mantener al menos 3 copias de tus datos (el original y dos copias).
  • 2 Medios Diferentes: Almacenar las copias en 2 tipos de medios diferentes (ej. disco duro interno y externo).
  • 1 Copia Fuera del Sitio: Mantener 1 de esas copias fuera de la ubicación física (ej. en la nube o en una ubicación remota).

Archivos .BAT para Automatización

Un archivo .bat (Batch) es un script simple para el sistema operativo que permite ejecutar una serie de comandos de manera secuencial, ideal para automatizar tareas repetitivas como el respaldo de archivos.

Ejemplo de script para respaldo usando XCOPY:

@echo off

xcopy "C:\Ruta\Origen" "D:\Ruta\Destino" /s /e /i /y

Switches del Comando XCOPY (Modificadores)

Estos parámetros indican a `xcopy` exactamente cómo debe realizar la copia de los directorios para asegurar que el respaldo sea completo y automático.

  • /s: Copia directorios y subdirectorios, **excepto** los que están vacíos.
  • /e: Se utiliza junto a `/s`. Fuerza la copia de **subdirectorios vacíos**, replicando toda la estructura del origen.
  • /i: Indica que el destino es un **directorio**. Si el directorio de destino no existe, `xcopy` lo crea automáticamente. (¡Clave para evitar errores!)
  • /y: **Suprime la confirmación** de sobrescritura. Es fundamental para la automatización, ya que evita que el script se detenga esperando una respuesta.

Programador de Tareas de Windows

Es la herramienta clave del sistema operativo para ejecutar programas o scripts (como un `.bat`) en momentos específicos o como respuesta a eventos definidos.

Funciones Clave para Backups

  • Activadores (Triggers): Define cuándo se ejecutará el script (ej. Diariamente a las 2:00 AM, al iniciar sesión).
  • Acciones: Especifica qué programa o script debe ejecutar (ej. `C:\Rutas\MiBackup.bat`).
  • Condiciones: Permite definir requisitos adicionales (ej. Solo ejecutar si la laptop está conectada a la corriente).
