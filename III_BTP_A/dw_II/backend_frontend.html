Introducción a la Clase / Front-end vs. Back-end

Introducción a Diseño Web II

Profundizar en conceptos y técnicas para la creación de sitios web dinámicos e interactivos.

  • Más allá del HTML y CSS estático.
  • Creación de experiencias web ricas y funcionales.

Concepto de Diseño Web

Proceso integral de crear sitios web: planificación, conceptualización, producción, mantenimiento.

  • Estética visual y usabilidad.
  • Funcionalidad y objetivos del sitio.

Front-end: La Parte Visible

Todo lo que el usuario final ve y con lo que interactúa directamente en un navegador web.

  • Se ejecuta en el dispositivo del cliente.
  • Es la "cara" de la aplicación web.

Tecnologías Front-end Clave

  • HTML: Estructura y contenido.
  • CSS: Estilos y presentación.
  • JavaScript: Interactividad y dinamismo.

Rol del Desarrollador Front-end

Implementar el diseño visual y asegurar una experiencia de usuario fluida.

  • Enfoque en Usabilidad, Accesibilidad, Responsividad.
  • Interfaz de Usuario (UI) y Experiencia de Usuario (UX).

Back-end: La Parte Invisible

Componentes que se ejecutan en el servidor: base de datos, lógica de aplicación.

  • Es el "cerebro" de la aplicación web.
  • Almacena, organiza y procesa datos.

Tecnologías Back-end Clave

  • Lenguajes de Servidor: Python, Node.js, PHP, Ruby, Java.
  • Bases de Datos: MySQL, PostgreSQL, MongoDB.
  • APIs: Comunicación entre aplicaciones.

Rol del Desarrollador Back-end

Construir y mantener la tecnología que alimenta la parte del servidor.

  • Enfoque en seguridad de datos, escalabilidad, eficiencia.
  • Lógica de negocio y gestión de datos.

Front-end vs. Back-end: La Conexión

Trabajan en conjunto para entregar una aplicación web completa.

  • Front-end envía solicitudes.
  • Back-end procesa y responde.
  • Interacción vía APIs.

Desarrollo Full-stack

Desarrollador con habilidades y experiencia tanto en Front-end como en Back-end.

Capaces de trabajar en todas las capas de una aplicación web.

Asignación de Tareas

Tarea: Mapa Conceptual de Componentes Web.

Los estudiantes deberán crear un mapa conceptual que ilustre los componentes clave del desarrollo web, diferenciando claramente entre Front-end y Back-end, y mostrando las tecnologías y roles asociados a cada uno. El mapa debe destacar las interconexiones.
