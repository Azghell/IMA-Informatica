¿Qué son los Eventos de JavaScript?

En JavaScript, un evento es una acción o suceso que ocurre en el sistema y que el navegador detecta. Estos eventos pueden ser provocados por el usuario o por el propio navegador, como un clic de ratón, la carga de una página o el envío de un formulario.

Manejadores de Eventos (Event Handlers)

  • Atributos HTML: Anticuado. `onclick="myFunction()"`. Mezcla la lógica con la estructura.
  • Propiedad del DOM: Asigna una función a la propiedad del evento. `element.onclick = myFunction;`. Permite un solo manejador por evento.
  • Método `addEventListener()`: La forma moderna y recomendada. Permite asignar múltiples funciones a un mismo evento. `element.addEventListener('click', myFunction);`.

Tipos de Eventos Comunes

  • Eventos de Ratón: `click`, `mouseover`, `mouseout`, `mousedown`, `mouseup`.
  • Eventos de Teclado: `keydown`, `keyup`, `keypress`.
  • Eventos de Formulario: `submit`, `change`, `focus`, `blur`.
  • Eventos de Carga: `load`, `unload`.

El Objeto Evento

Cuando un evento ocurre, el navegador crea un objeto `Event` que contiene información detallada sobre el evento, como el tipo de evento, el elemento que lo disparó, las coordenadas del ratón, etc. Este objeto se pasa como argumento a la función manejadora.

Bubbling y Capturing (Propagación de Eventos)

Bubbling (Burbujeo)

El evento se propaga desde el elemento más interno hasta el más externo. Es el comportamiento por defecto.

Capturing (Captura)

El evento se propaga desde el elemento más externo hasta el más interno. Menos común, se habilita con `true` en `addEventListener()`.

Detener la Propagación y el Comportamiento por Defecto

  • `event.stopPropagation()`: Detiene la propagación del evento, impidiendo que el evento `bubbling` o `capturing` afecte a otros elementos.
  • `event.preventDefault()`: Evita el comportamiento predeterminado del navegador, como el envío de un formulario o la navegación de un enlace.

Delegación de Eventos

La delegación de eventos es una técnica para manejar eventos en un contenedor padre en lugar de en cada elemento hijo individual. Es muy eficiente, especialmente para listas grandes o elementos creados dinámicamente.

Ejemplo: Evento `click`

Escucha un clic en un botón y muestra un mensaje.

<button id="myButton" class="bg-blue-500 text-white px-4 py-2 rounded-lg">Haz Clic Aquí</button>

<script>
  // Selecciona el botón por su ID
  const myButton = document.getElementById('myButton');

  // Añade un 'event listener' para el evento 'click'
  myButton.addEventListener('click', function() {
    alert('¡El botón ha sido clicado!');
  });
</script>

Ejemplo: Eventos de Ratón (`mouseover` y `mouseout`)

Cambia el color de un elemento cuando el cursor entra y sale de él.

<div id="hoverDiv" class="w-48 h-24 bg-blue-200 rounded-lg flex items-center justify-center transition-colors duration-200">
  Pasa el mouse aquí
</div>

<script>
  const hoverDiv = document.getElementById('hoverDiv');

  hoverDiv.addEventListener('mouseover', function() {
    hoverDiv.style.backgroundColor = '#3b82f6'; // Cambia a un azul más oscuro
  });

  hoverDiv.addEventListener('mouseout', function() {
    hoverDiv.style.backgroundColor = '#93c5fd'; // Vuelve al color original
  });
</script>

Ejemplo: Evento de Formulario (`submit`)

Maneja el envío de un formulario sin recargar la página.

<form id="myForm" class="flex flex-col space-y-2">
  <input type="text" placeholder="Escribe algo" class="border p-2 rounded">
  <button type="submit" class="bg-green-500 text-white px-4 py-2 rounded">Enviar</button>
</form>

<script>
  const myForm = document.getElementById('myForm');

  myForm.addEventListener('submit', function(event) {
    // Evita que el formulario se envíe y la página se recargue
    event.preventDefault();

    alert('El formulario ha sido enviado (sin recargar la página)');
  });
</script>
